HAPR probit

We have electronic health records on potential cardiac patients. The patients are followed for 10 years. We observe disease status (ASCVD) and a polygenic score. We also have covariates: PREVENT score (a standard risk estimate based on traditional risk factors) and gender.

We want to estimate the clinical utility of the polygenic score. And to estimate how this clinical utility is expected to improve as the polygenic score becomes more predictive. The current polygenic score is based on a GWAS with \(R^2\) of 2%, whereas the estimated heritability of ASCVD is around 20%. HAPR combines these heritability estimates with the current correlations to estimate a model of the future polygenic score.

The main estimation function is hapr().

library(hapr)

covariates <- heart_disease_data |> select(prevent_score, gender) # Covariates

hapr_fit <- hapr(
  y = heart_disease_data$ascvd, # Outcome
  gc = heart_disease_data$polygenic_score, # Polygenic score
  w = covariates, # Covariates
  model_type = "probit",
  r2_current = 0.02, # Current R-squared of the polygenic score
  r2_future = 0.2 # Future R-squared of the polygenic score, estimate of heritability
)

print(hapr_fit)

That was easy! Behind the scenes, hapr() run a feasible probit regression of ASCVD on the polygenic score and covariates. It then combines this with heritability estimates to estimate the “true model” of how ASCVD is predicted by the true polygenic score. The reported beta is the coefficient of the true polygenic score.

It is intuitive to see what is going on if we simulate the full model with the true polygenic score:

simulated_data <- hapr_simulate(hapr_fit, covariates)

The simulated_data contains the true polygenic score, the observed polygenic score, and the outcome. This is the dream dataset we would like to have to evaluate the utility of the polygenic score. Moreover, with the predict function, we can add the predictions from each model to the data.

risk_dataset <- predict(hapr_fit, simulated_data)

We can now compare how the current and future polygenic scores add value relative to the traditional risk factors. One simple way are scatterplots of the predicted risk using the polygenic score vs the predicted risk using the traditional risk factors. If the polygenic score adds little extra value, we should see the points clustered around the diagonal.

# Plot for current polygenic score
ggplot(risk_dataset, aes(x = y_hat_w, y = y_hat_gc_w)) +
  geom_point(alpha = 0.5) +
  geom_abline(slope = 1, intercept = 0, color = "red") +
  coord_fixed(ratio = 1) +  # Ensures 1:1 aspect ratio
  theme_minimal() +
  labs(x = "Predicted risk using traditional factors", 
       y = "Predicted risk using current polygenic score")
